# Anonymizer — техническая документация

🌐 **Русский** · [English](DOCUMENTATION_EN.md)

Локальный обезличиватель персональных данных (ПДн) для русскоязычного текста.
На входе строка — на выходе та же строка, где ПДн заменены на плоские метки
(`[Имя]`, `[Телефон]`, `[Адрес]`, …). Цель — безопасно отправлять обезличенный текст во
внешние облачные LLM, не передавая туда исходные данные.

## 1. Назначение и принципы

Anonymizer находит в русском тексте персональные данные (ФИО, телефоны, адреса, организации,
даты, суммы, email, ИНН, СНИЛС, паспорта, банковские карты, IP) и заменяет их на обезличенные
метки. Типичный сценарий: у вас есть конфиденциальный текст, который нужно отправить во внешнюю
облачную LLM (ChatGPT, Claude и т.п.) — сначала прогоняете его через Anonymizer, затем во вне
уходит уже безопасная версия без реальных данных.

Три принципа, на которых построен проект:

- **Необратимость.** Замена односторонняя: таблицы соответствий «метка → оригинал» не существует.
  Восстановить исходные данные по обезличенному тексту нельзя.
- **Полнота важнее точности (recall > precision).** Худший исход — пропустить ПДн. Перемаскировать
  лишнее (закрыть не-ПДн) допустимо. При сомнении система маскирует.
- **Строго локально.** Всё детектирование выполняется на вашей машине. Единственный сетевой
  вызов — к **локальному** серверу LM Studio (`localhost:1234`), который тоже работает у вас.
  Наружу ничего не уходит.

## 2. Требования

| Компонент | Назначение | Обязательность |
|---|---|---|
| Python 3.10+ | среда выполнения | обязательно |
| [Natasha](https://github.com/natasha/natasha) | NER + извлечение адресов/дат/сумм/ФИО | желательно |
| [Presidio](https://microsoft.github.io/presidio/) + spaCy `ru_core_news_lg` | NER + распознаватели РФ-ID | желательно |
| LM Studio + локальная LLM | семантический «добор» через локальную LLM | опционально |
| FastAPI + Uvicorn | HTTP API | опционально (только для API) |
| httpx | `TestClient` в тестах API | опционально (только тесты) |
| python-dotenv | чтение `.env` | опционально |

**Деградация по возможностям.** Любой недоступный компонент молча отключается, пайплайн продолжает
работать на оставшихся. Минимальная рабочая конфигурация — только regex-детектор (всегда доступен).
Полная — все 4 детектора + проверка LLM.

ОС: проект разрабатывался на Windows (примеры команд — PowerShell), но кросс-платформенный.

## 3. Установка

```bash
# 1. Склонировать репозиторий
git clone https://github.com/Kefis89/Anonimyzer.git
cd Anonimyzer

# 2. Создать виртуальное окружение
python -m venv .venv
# Windows (PowerShell):
.venv\Scripts\Activate.ps1
# Linux/macOS:
source .venv/bin/activate

# 3. Установить NLP-зависимости
pip install natasha presidio-analyzer
python -m spacy download ru_core_news_lg

# 4. (опционально) Зависимости HTTP API и тестов
pip install fastapi "uvicorn[standard]" httpx python-dotenv
```

> Версии не запинены (нет `requirements.txt`) — ставятся актуальные. Natasha тянет свои данные
> (navec/slovnet) пакетом. spaCy-модель `ru_core_news_lg` (~0.5 ГБ) скачивается отдельной командой.

> ⚠️ **setuptools должен оставаться < 81.** pymorphy2 (зависимость Natasha) ищет словари через
> `pkg_resources`, удалённый из setuptools ≥ 82 — с новым setuptools `MorphVocab()` падает и
> Natasha-детектор молча отключается. Если что-то обновило setuptools, откатите:
> `pip install "setuptools<81"`.

## 4. Настройка LM Studio + LLM

LLM-детектор и финальная проверка работают через **LM Studio** — локальный сервер с
OpenAI-совместимым API. Модель не зашита в код: используется та, что загружена в LM Studio.
Без LM Studio три остальных детектора работают как обычно (LLM-детектор просто отключается).

1. Установите [LM Studio](https://lmstudio.ai/).
2. Во вкладке поиска моделей скачайте подходящую инструкт-модель (требования — ниже).
3. Вкладка **Developer / Local Server** → загрузите модель и запустите сервер на порту **1234**
   (адрес по умолчанию `http://localhost:1234/v1`).
4. Готово. Anonymizer сам определит загруженную модель при импорте через `GET /v1/models`
   (берётся первая из списка; чтобы зафиксировать конкретную — задайте `LMSTUDIO["model"]`
   в `config.py`).

### Требования к модели

Детектор обращается к модели по OpenAI-совместимому `/chat/completions` и **не привязан к
конкретной модели**. Подойдёт любая, удовлетворяющая требованиям:

- **Structured output (`response_format: json_schema`).** В каждом запросе передаётся жёсткая
  JSON-схема с `enum` допустимых типов. Модель и раннер в LM Studio обязаны это поддерживать;
  иначе сервер вернёт HTTP 400, запрос упадёт, и LLM-слой тихо отключится (как при недоступном
  LM Studio) — пайплайн не падает, но добор и verify не работают. (`json_object` LM Studio **не**
  принимает.)
- **Инструкт/чат-модель**, следующая системному промпту; вывод детерминированный (`temperature = 0`
  выставляется кодом).
- **Хорошее понимание русского.** Промпты на русском; модель должна вернуть тип из русского набора
  (подстраховано `enum`) и **дословную** русскую подстроку. Если модель перефразирует или исказит
  подстроку, `text.find` её не найдёт и сущность молча отбросится — recall падает. Чем сильнее
  модель в русском, тем выше полнота.
- **Reasoning и обычные модели — обе подходят.** У reasoning-моделей JSON часто приходит в
  `reasoning_content` при пустом `content`; чтение сделано с запасным вариантом
  (`content or reasoning_content`).

### Примеры подходящих моделей

Любая современная инструкт-модель в LM Studio со structured output и приличным русским. Например:

- **Qwen2.5-7B-Instruct** (или Qwen3-8B) — именно на Qwen проект и тестировался; сильный русский.
- **Google Gemma 2 9B Instruct** — поддерживает structured output, неплохой русский.
- **Mistral-Nemo-Instruct (12B)** — мультиязычная, включая русский.

Более крупная/сильная модель обычно даёт выше recall (находит больше пропущенного), но работает
медленнее. Минимально рабочий вариант — 7–9B; для слабого железа берите квантованные сборки
(GGUF Q4/Q5).

### Прочее

- Если LM Studio подняли уже **после** старта Anonymizer, перечитайте модель: `POST /reload`
  (см. API) либо перезапустите процесс.
- Если сервер недоступен, при импорте быстро возвращается `connection refused`, LLM-детектор
  отключается, импорт не зависает.

## 6. Использование из стороннего Python-кода

Публичный API пакета:

```python
from anonymizer import anonymize, anonymize_detailed, AnonymizationResult

# 1. Простой вызов — только обезличенный текст
masked: str = anonymize("текст с ПДн")

# 2. Подробный вызов — текст + статистика по детекторам
res: AnonymizationResult = anonymize_detailed("текст с ПДн")
res.text          # обезличенный текст
res.spans_found   # dict: имя детектора → число СЫРЫХ находок (до слияния)
                  # напр. {"regex": 2, "natasha": 4, "presidio": 3, "llm": 1}
```

### Сигнатура `anonymize_detailed`

```python
anonymize_detailed(text: str, detectors=None, *, verify: bool = True) -> AnonymizationResult
```

- `detectors` — какие детекторы запускать. По умолчанию (`None`) — все включённые в
  `config.DETECTORS_ENABLED` (см. «Какие детекторы запускать» в разделе 10). Явно переданный
  список выполняется как есть, без фильтра по конфигу: передав один детектор, получите вклад
  ровно этого детектора.
- `verify` — выполнять ли финальную доп-проверку через LLM (`verify_with_llm`).

Запуск одного детектора в изоляции (без сети и без добора LLM):

```python
from anonymizer.pipeline import anonymize_detailed
from anonymizer.detectors import DETECTORS_BY_NAME

regex = DETECTORS_BY_NAME["regex"]
res = anonymize_detailed("ИНН 7707083893, e-mail a@b.com", [regex], verify=False)
# res.text == "ИНН [ИНН], e-mail [Email]"
```

### Важно: загрузка моделей при импорте (eager loading)

`import anonymizer` **медленный и имеет побочные эффекты**: модели Natasha/spaCy грузятся сразу,
а LLM-детектор делает сетевую пробу `localhost:1234`. Поэтому:

- Импортируйте пакет один раз при старте вашего приложения, а не на каждый запрос.
- Объекты не потокобезопасны — не вызывайте `anonymize_detailed` из нескольких потоков
  параллельно (в HTTP API запросы сериализуются `threading.Lock`).

### Какие типы маскировать

Маскируемые типы настраиваются константами в `anonymizer/config.py` (см.
[Конфигурация](#10-конфигурация)) — глобально для всего процесса.

## 7. HTTP API и запуск на сервере

Приложение FastAPI (`anonymizer/api.py`). Слушает `127.0.0.1:8077` по умолчанию (только локальный
доступ). Сервис принимает **необезличенный** текст, поэтому открывать его наружу (`0.0.0.0`) можно
только за reverse-proxy с TLS — см. «Развёртывание на сервере».

### Запуск

```powershell
.venv\Scripts\python.exe run.py
```

> `run.py` — основной файл запуска сервера (в корне проекта). Эквивалентная команда —
> `.venv\Scripts\python.exe -m anonymizer.api`.

### Переменные окружения

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `ANONYMIZER_HOST` | интерфейс прослушивания | `127.0.0.1` |
| `ANONYMIZER_PORT` | порт | `8077` |
| `ANONYMIZER_API_KEY` | ключ для заголовка `X-API-Key` | `change-me` |

Читаются из `.env` в корне проекта (если установлен `python-dotenv`) или из реальных переменных
окружения. **Обязательно задайте свой `ANONYMIZER_API_KEY`** — сгенерировать можно так:
`python -c "import secrets; print(secrets.token_hex(32))"`. Пока стоит дефолт `change-me` или
ключ короче 16 символов, сервер пишет предупреждение в лог. Файл `.env` в `.gitignore`;
готовый шаблон — `.env.example` в корне репозитория.

### Эндпоинты

| Метод/путь | Auth | Назначение |
|---|---|---|
| `GET /health` | нет | статус сервиса и активные детекторы |
| `POST /anonymize` | `X-API-Key` | полный ансамбль (включённые в `DETECTORS_ENABLED` + verify) |
| `POST /anonymize/{detector}` | `X-API-Key` | один детектор (`regex`/`natasha`/`presidio`/`llm`), без verify |
| `POST /reload` | `X-API-Key` | пере-определить модель LM Studio |

`POST /anonymize` и `/anonymize/{detector}` принимают `{"text": "..."}` и возвращают
`{"text": "...", "spans_found": {...}}`. У `/anonymize/{detector}` в `spans_found` ровно один ключ —
вклад выбранного детектора (удобно для анализа). Неизвестное имя детектора → HTTP 422.
Длина `text` ограничена `config.API["max_text_chars"]` (по умолчанию 100 000 символов,
сверх — HTTP 422): запросы обрабатываются по одному, гигантский текст заблокировал бы сервис.
Swagger UI (`/docs`, `/openapi.json`) отключён — лишняя поверхность для сервиса с ПДн.

Запросы **сериализуются** (`threading.Lock`): один за раз, потому что NLP-модели не
потокобезопасны (а LLM и так медленный). Сервер запускается с `workers=1`.

### Примеры обращения

```bash
# Проверка статуса (локально; при доступе по сети — https://ВАШ_ДОМЕН через TLS-прокси)
curl http://localhost:8077/health

# Обезличивание (полный ансамбль)
curl -X POST http://localhost:8077/anonymize \
  -H "X-API-Key: ВАШ_КЛЮЧ" -H "Content-Type: application/json" \
  -d '{"text":"Иванов Иван, ИНН 7707083893"}'

# Только один детектор
curl -X POST http://localhost:8077/anonymize/regex \
  -H "X-API-Key: ВАШ_КЛЮЧ" -H "Content-Type: application/json" \
  -d '{"text":"ИНН 7707083893"}'
```

Готовый Python-клиент на stdlib — `client_example.py` (функции `anonymize()`, `health()`).
Настраивается переменными окружения `ANONYMIZER_URL` (по умолчанию `http://localhost:8077`)
и `ANONYMIZER_API_KEY`. **Файл `.env` клиент не читает** — это пример внешней программы,
`.env` подхватывает только сервер. Если переменная не задана, клиент отправит дефолтный
ключ `change-me`, и сервер с настоящим ключом ответит 401:

```powershell
$env:ANONYMIZER_API_KEY = "<тот же ключ, что у сервера>"
.venv\Scripts\python.exe client_example.py
```

### Развёртывание на сервере

1. Склонируйте репозиторий и поставьте зависимости (раздел 3), включая `fastapi`/`uvicorn`.
2. Настройте LM Studio + LLM (раздел 4), если нужен 4-й детектор.
3. Создайте `.env` с боевым `ANONYMIZER_API_KEY` (и при необходимости `ANONYMIZER_PORT`).
4. **Для доступа по сети TLS обязателен**: сервис принимает исходный (необезличенный) текст и
   API-ключ, по открытому HTTP они уйдут в сеть в открытом виде. Поставьте reverse-proxy
   (nginx/caddy) с TLS перед сервисом, задайте `ANONYMIZER_HOST=0.0.0.0` (или адрес внутреннего
   интерфейса) и откройте в файрволе только порт прокси. Без TLS-прокси оставляйте дефолтный
   `127.0.0.1` — сервис будет доступен только с этой машины.

> **Подхват изменений конфига.** Сервер читает конфиг и грузит модели при импорте. После правок
> `config.py` (например, переключателей типов) **перезапустите процесс** — Uvicorn запущен без
> авто-reload.

## 8. Архитектура и пайплайн

### Модель данных — `Span`

Найденный фрагмент ПДн описывается `Span(start, stop, type, source)`:
полуинтервал `[start, stop)`, `type` — ключ из `LABELS`/`PRIORITY`, `source` — кто нашёл.

### Полный конвейер (`anonymizer/pipeline.py`, `anonymize_detailed`)

```
исходный текст
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│ 1. ДЕТЕКЦИЯ                                                  │
│    Каждый детектор из DETECTORS = [regex, natasha,           │
│    presidio, llm] возвращает список Span. Сырое число       │
│    находок каждого пишется в spans_found (ДО слияния).       │
└─────────────────────────────────────────────────────────────┘
      │
      ▼
  2. split_persons   — person-спан → [Имя]/[Фамилия]/[Отчество]
      │
      ▼
  3. фильтр типов    — выкинуть спаны типов, выключенных в
      │                ANONYMIZE_TYPES (до слияния)
      ▼
  4. merge_spans     — разрешить пересечения «закрашиванием» по
      │                PRIORITY + склейка фрагментов адреса
      ▼
  5. apply_spans     — заменить спаны метками (с конца строки,
      │                с обрезкой пробелов по краям)
      ▼
  6. verify_with_llm — LLM перечитывает закрашенный текст и
      │                 добирает оставшиеся ПДн (до max_verify_passes
      │                 раз); пропускается при verify=False
      ▼
  7. collapse_repeated_labels — схлопнуть подряд идущие ОДИНАКОВЫЕ
      │                          метки: "[X] [X]" → "[X]"
      ▼
обезличенный текст  (+ spans_found)
```

### Ключевые механизмы

- **Слияние «закрашиванием» (`merge_spans`).** Каждый символ достаётся типу с наибольшим
  `PRIORITY`, накрывающему его; затем смежные участки одного типа собираются в спаны. Так корректно
  разрешаются частичные пересечения. После этого соседние фрагменты `address`, разделённые лишь
  пунктуацией/пробелами, склеиваются в один (AddrExtractor дробит адрес на `г. Уфа` / `ул. Ленина`
  / `д. 5`).
- **Замена с конца (`apply_spans`).** Идём от конца строки, чтобы смещения не сбивались; пробелы
  по краям спана обрезаются, иначе соседние метки слипаются (`[Сумма][Дата]`).
- **Разбор персон (`split_persons`).** `NamesExtractor` ненадёжен на свободном тексте (путает
  должности с фамилиями), поэтому запускается **только на подстроке уже подтверждённого
  person-спана**. Если разобрать не удалось — остаётся `[ФИО]`.
- **Eager loading.** Модели загружаются при импорте модулей-детекторов
  (`natasha.NATASHA`, `presidio.PRESIDIO`, `llm.LMSTUDIO_MODEL` — глобали). Недоступный
  компонент → глобаль `None` → детектор молча выключен.

## 9. Детекторы

Четыре независимых детектора, зарегистрированы по порядку в `detectors/__init__.py`:
`DETECTORS = [regex, natasha, presidio, llm]`. Каждый — единообразный плагин: `NAME`,
`detect(text) -> list[Span]` (отказоустойчив, при недоступности `[]`), `available() -> bool`.
**Порядок важен**: при равном `PRIORITY` первый в списке «закрашивает» символы раньше, поэтому
структурные детекторы идут перед модельными.

### 9.1 regex — структурированные идентификаторы

Файл `detectors/regex.py`. **Всегда доступен.** Шаблоны в `config.REGEXES`. Ловит детерминированно:
email, телефоны РФ, банковские карты, СНИЛС, IPv4, паспорт, ИНН, денежные суммы.

Неочевидные правила (защита от ложных срабатываний — **не «чинить»**):
- **Паспорт требует разделитель** между серией и номером, иначе шаблон совпал бы с голым
  10-значным ИНН.
- **Сумма требует валюту** (символ/слово), иначе голые числа (ИНН, индексы) стали бы суммами.
- **Телефоны** ловятся в трёх формах: с префиксом `+7`/`8`; без префикса, но с обязательными
  разделителями; 7-значный локальный.

### 9.2 natasha — NER + правила

Файл `detectors/natasha.py`. NER-теги PER/LOC/ORG → `person`/`loc`/`org`; плюс rule-based
экстракторы: `AddrExtractor` → `address`, `DatesExtractor` → `date`, `MoneyExtractor` → `money`.
`NamesExtractor` используется отдельно — в `split_persons` для разбора ФИО.

### 9.3 presidio — spaCy + распознаватели РФ-ID

Файл `detectors/presidio.py`. NER на spaCy `ru_core_news_lg` + кастомные шаблонные распознаватели
`RU_INN` / `RU_SNILS` / `RU_PASSPORT` + встроенные CreditCard/Ip, явно зарегистрированные
для `ru`.

> **`PhoneRecognizer` удалён из реестра.** Библиотека `phonenumbers` жадно принимает голые группы
> цифр (ИНН/паспорт) за телефон, а телефон по приоритету выше ИНН → получался `[Телефон]` на месте
> `[ИНН]`. РФ-телефоны надёжно ловит regex-детектор.

> **`EmailRecognizer` не регистрируется.** Его `validate_result` вызывает `tldextract`, который
> при пустом кэше делает HTTPS-запрос в интернет за public suffix list — это нарушает принцип
> «строго локально» (а без интернета даёт зависание на таймауте). Email надёжно ловит
> regex-детектор с тем же приоритетом 100.

### 9.4 llm — семантический добор через LLM

Файл `detectors/llm.py`. Локальная LLM через LM Studio. Роль — **контекстный добор
смысловых сущностей** (имена, организации, локации, адреса, даты, суммы), которые пропустили
формальные детекторы.

**Как работает (важно для понимания гарантий):**
- LLM **не возвращает** ни готовую строку, ни координаты — он отдаёт список
  `{"type": "...", "text": "<дословная подстрока>"}`.
- Спаны строит сам Anonymizer: ищет дословную подстроку в оригинале (`text.find`) и помечает все
  её вхождения. Если модель «придумала» подстроку, которой нет в тексте — она просто **не
  находится и отбрасывается**. Изменить, вставить или удалить символы в исходном тексте LLM
  **не может** (см. [раздел 13](#13-гарантии-и-ограничения)).
- **Структурированные ID LLM не возвращает** (`config.LLM_SKIP_TYPES`: email/phone/card/ip/
  passport/snils/inn) — на них он часто ошибается (путает СНИЛС с телефоном, ИНН с паспортом), а
  владеют ими regex/Presidio.
- **Проверочный проход** (`verify_with_llm`): LLM перечитывает уже закрашенный текст, игнорируя
  метки `[...]`, и ищет оставшиеся ПДн — до `max_verify_passes` раз.

### Приоритеты при конфликте (`config.PRIORITY`)

Чем больше число — тем важнее метка на пересечении:

| Приоритет | Типы |
|---:|---|
| 100 | email, phone, card, ip, passport, snils |
| 95 | inn |
| 90 | surname, name, patronymic |
| 80 | address |
| 70 | person |
| 60 | money |
| 50 | org |
| 40 | loc |
| 30 | date |

### Метки (`config.LABELS`)

`name`→`[Имя]`, `surname`→`[Фамилия]`, `patronymic`→`[Отчество]`, `person`→`[ФИО]`,
`org`→`[Организация]`, `loc`→`[Локация]`, `address`→`[Адрес]`, `email`→`[Email]`,
`phone`→`[Телефон]`, `inn`→`[ИНН]`, `snils`→`[СНИЛС]`, `passport`→`[Паспорт]`,
`card`→`[БанковскаяКарта]`, `ip`→`[IP]`, `date`→`[Дата]`, `money`→`[Сумма]`.

## 10. Конфигурация

Всё «крутибельное» собрано в `anonymizer/config.py`, отдельно от логики.

| Объект | Что задаёт |
|---|---|
| `LABELS` | тип → текст метки |
| `PRIORITY` | кто побеждает на пересечении |
| `REGEXES` | шаблоны структурных ID (блоки `_MONEY_*` собирают шаблон суммы) |
| `LMSTUDIO` | `base_url`, `model` (None = автоопределение), `timeout`, `temperature`, `max_tokens`, `max_verify_passes` |
| `LLM_SKIP_TYPES` | какие типы НЕ брать из ответа LLM |
| `ANONYMIZE_*` / `ANONYMIZE_TYPES` | какие типы маскировать |
| `DETECTOR_*` / `DETECTORS_ENABLED` | какие детекторы запускать |
| `API` | host / port / api_key (переопределяемы через env) / max_text_chars |

### Какие типы обезличивать

На каждый тип спана есть булева константа (по умолчанию `True` для всех, кроме `money` —
суммы осознанно не маскируются):

```python
ANONYMIZE_NAME = True
ANONYMIZE_EMAIL = True
ANONYMIZE_MONEY = False   # суммы НЕ маскируем (дефолт проекта)
...
```

Они собираются в словарь `ANONYMIZE_TYPES`, а пайплайн проверяет `is_type_enabled(type)` и
**отбрасывает спаны выключенных типов до слияния** — и в основном проходе, и в `verify_with_llm`.

Поведение:
- Выключенный тип **не маскируется**, но **по-прежнему обнаруживается** (`spans_found` не меняется —
  это статистика детекции, не маскирования).
- Компоненты ФИО управляются по отдельности: можно выключить `name`, оставив `surname`/`patronymic`
  (фильтр стоит после разбора персон).
- Для HTTP API правки конфига подхватываются только после перезапуска процесса.

### Какие детекторы запускать

Аналогично типам, на каждый детектор есть булева константа (по умолчанию все `True`):

```python
DETECTOR_REGEX = True
DETECTOR_NATASHA = True
DETECTOR_PRESIDIO = True
DETECTOR_LLM = True
```

Они собираются в словарь `DETECTORS_ENABLED`; проверка — `is_detector_enabled(name)`.

Поведение:
- Выключенный детектор **не запускается** в наборе по умолчанию (`anonymize(text)`,
  `POST /anonymize`) и **не появляется в `spans_found`**.
- Фильтр действует только на запуск без явного списка: `anonymize_detailed(text, [module])`,
  `POST /anonymize/{detector}` и eval-скрипты запускают переданный детектор в любом случае
  (диагностика и оценка детекторов продолжают работать).
- `DETECTOR_LLM = False` дополнительно отключает финальную LLM-проверку (`verify_with_llm`) —
  то есть LLM не вызывается вообще.
- Выключение детектора **не отменяет загрузку его моделей при импорте** (eager loading); оно
  убирает только его участие в конвейере.
- Как и остальной конфиг, подхватывается после перезапуска процесса.

## 11. Инструменты оценки детекторов

Отдельные скрипты в папке `scripts_for_testing/` для сравнения работы детекторов на корпусе. Корпус
`examples.json` лежит там же, а все генерируемые `.json`/`.html` результаты — в подпапке
`scripts_for_testing/results/` (создаётся автоматически). Запускаются интерпретатором venv. Две
цепочки:

> **Все примеры в `examples.json` и `examples_en.json` сгенерированы нейросетью** — это вымышленные
> тексты без реальных персональных данных; любые совпадения с настоящими людьми, адресами или
> реквизитами случайны.

**Матрица «пример × детектор»:**
```powershell
.venv\Scripts\python.exe scripts_for_testing\compare_all_detectors.py   # examples.json → results.json
.venv\Scripts\python.exe scripts_for_testing\make_compare_report.py   # results.json → report.html
```
`compare_all_detectors.py` гоняет каждый пример из `examples.json` через каждый детектор по отдельности
(`anonymize_detailed(text, [module], verify=False)`) и пишет `results.json` (для каждой пары:
`spans_found`, `labels_in_text`, обезличенный `text`). `make_compare_report.py` строит из этого
самодостаточную HTML-таблицу. По завершении `compare_all_detectors.py` сам вызывает
`make_compare_report.py`, поэтому `report.html` обновляется автоматически — отдельная команда выше
нужна лишь чтобы перестроить отчёт без повторного прогона.

**Оценка вклада LLM (самого медленного детектора):**
```powershell
.venv\Scripts\python.exe scripts_for_testing\compare_llm.py [N]    # examples.json → llm_impact.json
.venv\Scripts\python.exe scripts_for_testing\make_llm_report.py    # llm_impact.json → llm_report.html
```
`compare_llm.py` прогоняет каждый пример в двух конфигурациях — база `[regex, natasha, presidio]`
(без LLM, `verify=False`) против всех 4 + verify — фиксируя число спанов, число замен, **время** и
совпадение результирующих текстов. Необязательный `[N]` ограничивает число примеров (LLM делает до
3 запросов к LM Studio на пример; полный прогон ~100 примеров занимает 20+ минут).
`make_llm_report.py` строит сравнительную таблицу с колонкой «Результат совпадает». По завершении
`compare_llm.py` сам вызывает `make_llm_report.py`, поэтому `llm_report.html` обновляется
автоматически — отдельная команда выше нужна лишь чтобы перестроить отчёт без повторного прогона.

> Соглашение: **runner'ы** (`compare_all_detectors.py`, `compare_llm.py`) импортируют `anonymizer` — лежат в
> подпапке, поэтому добавляют корень проекта в `sys.path` (`parents[1]`); грузят модели, LLM
> требует LM Studio. **Генераторы отчётов** (`make_compare_report.py`, `make_llm_report.py`) — только stdlib
> и **не импортируют** `anonymizer` (читают JSON → быстро, без загрузки моделей). Все пути берутся
> относительно скрипта (`Path(__file__).parent`): вход `examples.json` — в `scripts_for_testing/`,
> а генерируемые `.json`/`.html` — в `scripts_for_testing/results/`.

## 12. Тесты

Тесты лежат в `tests/`; корневой `conftest.py` добавляет корень проекта в `sys.path`, чтобы
`import anonymizer` работал и при `pytest`, и при `python -m pytest` из любого каталога.

```powershell
# Весь набор
.venv\Scripts\python.exe -m pytest -v

# Один файл / класс / кейс
.venv\Scripts\python.exe -m pytest tests/test_anonymizer.py -v
.venv\Scripts\python.exe -m pytest tests/test_anonymizer.py::TestAnonymizeTypeConfig -v
.venv\Scripts\python.exe -m pytest tests/test_anonymizer.py::TestRegexStructuredIDs::test_email -v
```

- Юнит-тесты чистых функций (regex / merge / apply / `parse_entities` / схлопывание / переключатели
  типов и детекторов) работают без опциональных зависимостей.
- Тесты, которым нужна Natasha/Presidio, помечены `skipif` (`needs_natasha`/`needs_presidio`) — в
  regex-only среде они пропускаются, а не падают.
- Autouse-фикстура глобально отключает LLM/LM Studio (`llm.LMSTUDIO_MODEL = None`) → тесты
  детерминированы и не ходят в сеть; разбор ответа LLM проверяется отдельно на фиксированном JSON.

## 13. Гарантии и ограничения

- **Необратимость.** Нет таблицы соответствий; восстановить оригиналы по обезличенному тексту
  нельзя.
- **Полнота важнее точности.** Возможно перемаскирование (закрытие не-ПДн) — это осознанный
  компромисс в пользу непропуска данных. Особенно LLM может вернуть лишнюю/частую подстроку, и
  тогда замаскируются все её вхождения.
- **LLM не искажает исходный текст.** Модель не формирует выходную строку — она лишь называет, что
  маскировать, дословной цитатой. Замену делает детерминированный код на оригинале. «Галлюцинация»
  подстроки, которой нет в тексте, безвредна — она не находится и отбрасывается.
- **Не потокобезопасно.** NLP-модели не рассчитаны на параллельный вызов; HTTP API сериализует
  запросы `threading.Lock`. Для масштабирования — несколько экземпляров за балансировщиком.
- **Только локально.** Единственный внешний вызов — к локальному LM Studio. Если он выключен, LLM
  тихо отключается, остальные детекторы работают.
- **Качество детекции** зависит от моделей (Natasha/spaCy/LLM) и шаблонов regex — это не
  юридически сертифицированное средство обезличивания, а инженерный фильтр.
